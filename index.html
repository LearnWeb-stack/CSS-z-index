CSS Z-Index & Stacking

Master element layering and stacking contexts

🎭 Introduction to Z-Index

The z-index property controls the stacking order of positioned elements. Think of it as controlling which elements appear in front of or behind others, like layers in a drawing application.

Key Concepts

  • Stacking Order: The visual layering of elements on the z-axis
  • Stacking Context: A 3D conceptualization of HTML elements
  • Z-Index Values: Integers that determine layer priority
  • Positioned Elements: Only positioned elements can use z-index

📍 Important: Z-index only works on positioned elements (position: relative, absolute, fixed, or sticky). Static elements ignore z-index values.

🔢 Basic Z-Index Values

Z-index accepts integer values (positive, negative, or zero). Higher values appear in front of lower values.

.element-behind { position: relative; z-index: 1; /* Behind */ } .element-middle { position: relative; z-index: 2; /* Middle */ } .element-front { position: relative; z-index: 3; /* Front */ }

⬇️ Negative Z-Index

Negative z-index values place elements behind the default stacking layer, useful for background effects.

.background-element { position: absolute; z-index: -1; /* Behind normal content */ background: rgba(0,0,0,0.7); } .normal-content { position: relative; z-index: 1; /* In front of background */ }

🏗️ Understanding Stacking Contexts

A stacking context is a three-dimensional conceptualization of HTML elements. Elements within the same stacking context are layered according to their z-index values.

/* Parent creates stacking context */ .parent { position: relative; z-index: 1; } /* Child z-index is relative to parent's context */ .child { position: absolute; z-index: 999; /* Only competes with siblings */ }

What Creates a Stacking Context:

  • Root element (html)
  • Elements with position and z-index other than auto
  • Elements with opacity less than 1
  • Elements with transform, filter, or backdrop-filter
  • Elements with mix-blend-mode other than normal
  • Flex/Grid items with z-index other than auto
  • Elements with isolation: isolate

⚠️ Critical Rule: Child elements can never appear above their parent's stacking context, regardless of their z-index value.

🔗 Related Properties

Transform and Z-Index

Elements with CSS transforms create new stacking contexts, which can affect z-index behavior.

.transformed-element { transform: rotate(45deg); /* Creates stacking context */ z-index: 1; }

Opacity and Stacking

Elements with opacity less than 1 create stacking contexts.

.semi-transparent { opacity: 0.7; /* Creates stacking context */ z-index: 2; }

Isolation Property

The isolation property can create stacking contexts without side effects.

.isolated { isolation: isolate; /* Creates clean stacking context */ }

🎨 Common Z-Index Patterns

Modal and Overlay System

/* Common z-index scale for web applications */ .page-content { z-index: 1; } .dropdown-menu { z-index: 100; } .sticky-header { z-index: 200; } .modal-backdrop { z-index: 1000; } .modal-content { z-index: 1001; } .tooltip { z-index: 1100; } .notification { z-index: 1200; }

CSS Custom Properties for Z-Index

:root { --z-behind: -1; --z-default: 0; --z-content: 1; --z-header: 10; --z-dropdown: 100; --z-modal: 1000; --z-tooltip: 1100; } .modal { z-index: var(--z-modal); }

Flexbox and Grid Z-Index

.flex-container { display: flex; } .flex-item { z-index: 1; /* Works on flex items */ } .grid-container { display: grid; } .grid-item { z-index: 2; /* Works on grid items */ }

✨ Best Practices

🎯 Z-Index Management Tips:

  • Use a scale: Stick to multiples of 10 or 100 for easy insertion
  • Document your system: Create a z-index reference guide
  • Start small: Don't jump to 9999 immediately
  • Group by function: Keep related elements in similar ranges
  • Use CSS variables: Centralize z-index values

⚠️ Common Pitfalls:

  • Forgetting that z-index only works on positioned elements
  • Not understanding stacking context isolation
  • Using extremely high values (999999) unnecessarily
  • Fighting stacking contexts instead of working with them
  • Not testing across different browsers and devices

🔧 Debugging Z-Index Issues:

  • Use browser dev tools to inspect stacking contexts
  • Add temporary background colors to see layer boundaries
  • Check for unintended stacking context creation
  • Verify element positioning (static elements ignore z-index)
  • Test with CSS transforms and opacity effects
💡 Pro Tip: When debugging z-index issues, look for elements that unintentionally create stacking contexts (transforms, opacity, filters, etc.).
